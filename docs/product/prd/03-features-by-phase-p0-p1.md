# 03-分阶段功能需求（P0/P1）

## 4. 功能需求

### 4.1 功能分组

#### 关键词与来源配置

- 管理监控关键词
- 支持关键词创建、更新、删除和启停
- 管理基础来源启停和来源标识
- 管理简单设置项，如邮件通知开关、检查频率、默认来源列表

#### 热点发现

- 多源内容采集
- 内容标准化为统一热点候选
- URL + 来源维度去重
- AI 查询扩展
- AI 真实性判断、相关性评分、关键词命中判断、重要性判断和摘要

#### 通知与触发

- 手动触发热点检查
- 简单定时触发热点检查
- 邮件通知
- 邮件未配置时降级为仅站内/控制台展示

#### 控制台浏览

- 热点列表
- 热点事件详情
- 来源追踪
- 按关键词、来源、重要性、时间范围筛选
- 按热度/相关性/时间排序

#### 后续反馈

- 按关键词搜索事件
- 收藏、忽略、误报标记（反馈闭环）

### 4.2 优先级与 MVP 交付边界

#### P0（本轮必须）

- 关键词配置、编辑、删除和启停
- 至少两个来源的内容抓取与标准化
- URL + 来源维度去重
- AI 查询扩展
- AI 真实性判断、相关性评分、相关性理由、关键词是否命中、重要性和摘要
- 热点列表、详情、来源追踪
- 按来源、重要性、时间、关键词筛选
- 手动触发热点检查
- 简单定时触发热点检查
- 邮件通知可选配置
- 邮件缺失时系统可继续运行

#### P1（阶段推进）

- 重点账号管理
- 更多数据源接入
- WebSocket 或等价实时推送
- 人工反馈的持续优化作用
- 日报增强
- X 趋势词监测
- 更细粒度筛选能力

#### P2（后续迭代）

- 多用户、多租户与权限
- 企业微信 / Telegram / 飞书推送
- 实时预警策略
- 历史趋势图
- 个性化推荐
