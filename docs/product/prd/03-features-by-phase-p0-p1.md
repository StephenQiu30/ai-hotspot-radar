# 03-分阶段功能需求（P0/P1）

## 4. 功能需求

### 4.1 功能分组

#### 来源治理

- 管理采集来源
- 配置来源启停、频率、权重、语言和地区
- 管理 X 关键词规则和重点账号名单

#### 热点发现

- 多源内容采集
- 内容标准化
- 去重与事件聚类
- 热度评分（跨源证据为主）

#### 日报生成

- AI 中文摘要
- 热点排序
- 每日热点日报编排
- 邮件投递

#### 控制台浏览

- 今日热点榜
- 热点事件详情
- 来源追踪
- 主题与来源筛选

#### 搜索与反馈

- 按关键词搜索事件
- 按来源、时间、主题筛选
- 收藏、忽略、误报标记（反馈闭环）

### 4.2 优先级与 MVP 交付边界

#### P0（本轮必须）

- 多来源配置管理
- 内容采集与标准化
- 去重与事件聚类
- 热度评分
- AI 中文摘要与日报生成
- 邮件推送
- 控制台热点榜
- 热点详情与来源追踪
- 搜索与筛选

#### P1（阶段推进）

- 人工反馈的持续优化作用
- 重点账号管理
- 关键词规则管理
- X 趋势词监测
- 更细粒度筛选能力

#### P2（后续迭代）

- 企业微信 / Telegram 推送
- 实时预警
- 历史趋势图
- 多租户与权限
- 个性化推荐
