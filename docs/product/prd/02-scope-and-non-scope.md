# 02-范围与非目标

## 3. 产品范围

### 3.1 本期范围（MVP）

- 完整中文产品与技术文档体系（PRD 与执行文档已结构化）
- 交付规范化项目骨架
- 围绕“日报 + 控制台 + 多源热点聚合”设计产品能力
- 覆盖中英双语 AI 圈公开信息
- 推送渠道先以邮件为主
- 数据源优先接入 API、RSS 与官方 X API
- 仅实现“监听、聚合、归档、邮件投递、控制台可查”闭环

### 3.2 非目标

- 不做多租户、组织权限、计费与商业化系统
- 不做完整第三方正文镜像站
- 不做高成本全网社交媒体 firehose
- 不做视频平台与封闭平台的复杂采集链路
- 不追求一次性覆盖“所有平台实时特征检测”

## 3.3 外部平台接入边界（含 X）

- X 为核心来源之一，但不能单独决定热点总榜
- X 采集优先官方 API 方式
- 不将未验证的第三方抓取作为第一阶段主链路
- 优先接入具备公开结构化输出的来源
- 控制范围内的来源优先级调整应统一在来源治理条目中管理

## 3.4 关键约束

- 摘要必须保留来源证据链
- 不默认长期保存完整第三方文章正文
- 技术实现遵循 `OpenAPI First` 与分层模块化
