# 05-风险与假设

## 6. 风险与假设

### 6.1 关键风险

- 第三方 API 或公开页面限制导致采样覆盖不足
- 抓取源偶发不稳定，可能引入噪声或漏检
- AI 分析可能误判真实性或相关性，需要保留来源与理由供复核
- 单一热点过度来自 X/Twitter，导致误判概率上升
- 邮件投递依赖外部服务可用性导致送达不稳定
- 开源用户的 API Key、网络环境和邮箱配置差异较大，必须支持缺省降级
- 真实外部来源和模型调用需要凭据与网络条件，自动化测试只能覆盖代码路径与降级行为

### 6.2 主要假设

- 首轮以单实例自部署为主，不做多租户与复杂权限体系
- 手动触发 + 简单定时触发可以满足 MVP 验证
- 邮件通知是可选能力，未配置不影响热点发现和控制台浏览
- 采用分阶段优化策略：先保证链路稳定，再逐步引入更细调参机制
- 数据源以公开、低门槛、可自部署用户容易配置的来源为主，不追求全网覆盖
- 日报/周报以本地模板为 P0 基线，AI 报告增强不阻塞 MVP 验收
