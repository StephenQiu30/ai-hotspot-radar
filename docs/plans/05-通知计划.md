# 05 Notification Plan

## 目标

实现 SMTP 热点事件邮件、日报/周报邮件和通知状态记录。

## 范围

- 新建 `notifications` 表。
- 使用 SMTP 发送热点事件邮件。
- 使用 SMTP 发送日报/周报邮件。
- SMTP 未配置时跳过邮件发送，但主流程继续。
- 记录发送成功、失败、跳过状态。

## 数据模型

- `id`
- `hotspot_id`
- `report_id`
- `channel`
- `recipient`
- `status`
- `error_message`
- `sent_at`
- `created_at`

## 验收标准

- SMTP 配置存在时可以发送邮件。
- SMTP 配置缺失时任务继续，并记录 skipped。
- SMTP 发送失败时记录 error_message。
- 邮件内容包含标题、摘要、来源链接和相关性理由。
- 低相关 `filtered` 热点不发送事件邮件。
- 日报/周报发送记录必须关联 `report_id`。

## 非目标

- 不做企业微信、飞书、Telegram。
- 不做复杂通知策略。
