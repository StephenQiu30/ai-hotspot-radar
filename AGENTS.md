# AGENTS

本文件定义本仓库文档与实现协作约束。

## 1. 文档主事实源

- 产品需求与范围以 `docs/product/prd.md` 及其子文件为准。
- 执行计划以 `docs/product/plan.md` 为准。
- 技术约束以 `docs/engineering/tech-spec.md` 为准。
- 接口与契约以 `contracts/openapi/openapi.yaml` 为准。

## 2. 改动规范

- 新增需求必须先落 PRD 对应章节，再映射到技术文档/验收项。
- 不得在未更新 PRD 的情况下扩展新来源、规则或能力范围。
- 不鼓励一次性重构：优先“可运行 + 可回滚”的小步迭代。
- 每次实现改动应可追踪到某个 PRD P0/P1 条目。
- 新增字段或新增 API 时，必须同步更新 OpenAPI、主技术文档和验收项，不得仅在代码层面变更。

## 3. 代码与测试

- Python 代码保持分层清晰：`services` 仅做装配，`backend/core` 保持领域与应用边界。
- 新增行为默认配套测试；核心流程至少覆盖正向主链路。
- 文档更新、API 更新、代码提交应同步进行，不出现“先写代码后补文档”。

## 4. 运维与交付

- 依赖与凭据仅允许通过环境变量注入。
- 邮件与定时链路可观测：失败场景要有可追踪状态记录。
- 评审时优先看“问题可运行性与回归成本”，其次看设计优雅性。
